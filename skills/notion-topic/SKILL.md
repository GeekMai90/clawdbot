# notion-topic

## 触发方式

麦先生发送以下格式的消息：
- `选题 <想法内容>`
- `添加选题 <想法内容>`

## 工作流程

1. 提取想法内容（去掉触发词后的所有文本）
2. 根据想法自动拟一个简洁的选题标题（10-30字）
3. 在 Notion「内容选题库」数据库中创建新页面：
   - **选题标题**：AI 生成的标题
   - **状态**：灵感
   - **优先级**：中（默认）
   - **简介**：想法内容的前 100 字摘要
4. 在页面正文中写入完整想法内容（带「💡 选题想法」标题）
5. 回复确认 + Notion 页面链接

## Notion 配置

- **数据库 ID**: `312b4958-904e-8031-9c59-000b830ad1bd`
- **数据库名称**: 内容选题库 Content Topics
- **API Key**: `~/.config/notion/api_key`
- **API 版本**: `2025-09-03`

## 字段说明

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 选题标题 | title | AI 生成 | 根据想法内容自动拟定 |
| 状态 | status | 灵感 | 可选：灵感/调研/待写/写作中/已发布/归档 |
| 优先级 | select | 中 | 可选：高/中/低 |
| 正文 | blocks | 完整想法 | 页面正文，带「💡 选题想法」标题 |

## 使用示例

**输入：**
```
选题 最近发现 OpenClaw 的书签整理功能特别好用，可以写一篇教程，
介绍如何用 AI 自动分类书签、补全 tags 和 alias，还能定时整理收集箱。
适合知识管理爱好者和效率工具玩家。
```

**输出：**
```
✅ 选题已添加到 Notion！

📝 OpenClaw 书签整理功能实战教程
🔗 https://www.notion.so/...

状态：灵感 | 优先级：中
```

## 实现细节

1. **标题生成规则**：
   - 简洁明确，10-30 字
   - 体现核心主题
   - 避免过于宽泛或过于细节
   - 可以带数字、emoji（适度）

2. **正文格式**：
   ```
   ## 💡 选题想法
   
   [麦先生的完整想法原文]
   ```

3. **错误处理**：
   - API Key 不存在 → 提示配置
   - 数据库无权限 → 提示分享给 integration
   - 网络错误 → 提示稍后重试
